Appendix: Getting Your Degree

"The best way to learn programming is to read and write code."
My CS Professor

Programming can be a lucrative profession, and it also makes a wonderful hobby. It is not necessary to get a degree in computer science to land a job as a programmer: The rule of thumb is that two good projects are worth an entire CS degree. In this addendum, we will discuss what must and what should be learned to become an good programmer.

Languages

There are a bazzillion programming languages in use today, and you're free to do whatever you want. Some people find sport in writing programs in the language Brainfuck, a language intentionally obtuse (to the extreme). I'm only going to cover three languages: Two that I think you must know, and one that I think should know.

Python is a powerful, general-purpose interpreted language, and makes a wonderful first language. It's in wide use by the scientific community, and is even available on the latest generation of TI-84 Graphing calculators!

Pro Tip:

Ignore C++

The C programming language is the lingua franca of computing, and any developer worth their salt knows their way around it. As of this writing, it is being relegated to use at the assembly level, with other languages replacing it for application development. Learning it will expose you to fundamental ideas in computing, such as memory addresses and numeric interpretation of bytes. The standard text for this language is The C Programming Language, affectionately referred to as K&R.

The final programming language worth learning (in my opinion) is Rust. The goal of the Rust team is to empower everyone to build reliable and efficient software, and they have excelled. Rust is fast, memory-safe and has been lauded as developer's Most-Loved language for six years running.[a] While it's a great language, it is not easy to learn! If you stumble through learning it as I did, try to keep in mind that What you are doing is not simple.

You can create great things without the above languages. Github, for example, was created by a team of Ruby developers. The three languages given above are simply staples of the modern development scene: Anything you could want to do with a computer, you could do with one of the three.

Standard Courses

After learning a programming language or two, data structures are the next item to check off in becoming a developer. All modern standard libraries include smart implementations of common data structures, so that learning how they work is a bit of an academic exercise— but worthwhile, for sure. The developer of today needs to know when to use what, but by and large, does not need to know their in's and out's.

The best text on data structures that I have found is Open Data Structures by Pat Morin. Advanced studies delve further into mathematics, usually focusing on analysis of algorithms. The canonical text of algorithms is Introduction to Algorithms.

Discrete Mathematics

Given the inherently discrete nature of the bit, discrete mathematics founds much of (formal) computer science— Most development is comparatively informal, though! It should be stated explicitly that experience in programming always wins out over academic studies, in much the same way that practice always wins out over theory.

Discrete mathematics is not a difficult subject, on par with a first semester in calculus. Most courses introduce enough formality to build proofs, then cover graphs and counting techniques, I liked Discrete Mathematics - An Open Introduction

Computer Architecture

Ah, computer architecture. I loved studying this subject, but honestly wonder how useful it is. It looks into the abstractions presented by hardware itself, such as signed and unsigned numbers, and instruction set architectures. A good development of this material is highly mathematical. I liked Computer Systems: A Programmer's Perspective.

Operating Systems

A very large topic, it discusses operating systems in the abstract. A good, if lighthearted, development is given by Remzi H. Arpaci-Dusseau and Andrea C. Arpaci-Dusseau in their free Operating Systems: Three Easy Pieces.

Aside:

In two words: Helper Functions!

Write code that can be improved. This just means that the developer's first responsibility is to keep the code simple.
  • Keep it simple

Code is read much more often than it is written. As such, it is the responsibility of the developer to write code which can be read easily— that is, which is clear.

If you do wish to contribute a modification to a project, then you have a few responsibilities:

  • Write code that can be read easily
    • Code is read much more often than it is written
  • Write code that is modular
    • Account for future development
But most of all, do one thing and do it well.
